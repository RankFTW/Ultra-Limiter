Ultra Limiter v2.0 — Code Origins & Provenance (Clean-Room Rewrite)
====================================================================

This version of Ultra Limiter is a CLEAN-ROOM REWRITE. All code was
written from scratch using ONLY public API documentation as reference.
No code from Display Commander, Special K, or any other frame limiter
project was used, copied, or referenced during development.

CLEAN-ROOM METHODOLOGY
=======================

The rewrite was performed by writing each module from its public API
documentation only:

  - NVIDIA NVAPI SDK (MIT license): nvapi.h for struct definitions
    (NV_SET_SLEEP_MODE_PARAMS, NV_LATENCY_MARKER_PARAMS, etc.),
    nvapi_interface.h for QueryInterface function IDs.

  - ReShade Addon API (BSD-3 license): reshade.hpp for register_addon,
    addon_event callbacks, register_overlay.

  - Dear ImGui (MIT license): imgui.h for UI widgets (SliderInt,
    Checkbox, BeginCombo, GetForegroundDrawList, AddText, AddLine).

  - MinHook (BSD-2 license): MinHook.h for MH_CreateHook, MH_EnableHook,
    MH_DisableHook, MH_RemoveHook.

  - Windows API (MSDN documentation): QueryPerformanceCounter/Frequency,
    CreateWaitableTimerEx (CREATE_WAITABLE_TIMER_HIGH_RESOLUTION),
    ZwSetTimerResolution/ZwQueryTimerResolution (ntdll),
    WritePrivateProfileString/GetPrivateProfileString,
    EnumDisplayMonitors, SetWindowPos, GetWindowLong/SetWindowLong,
    CreateThread, LoadLibraryA, GetProcAddress, GetModuleHandleW.

No source code from Display Commander was open during the rewrite.
The feature specification was derived from the Ultra Limiter v1.0
configuration struct (which documents WHAT the addon does, not HOW).


FILE-BY-FILE BREAKDOWN
======================

ul_timing.hpp / ul_timing.cpp
------------------------------
Origin: Clean-room from Windows API documentation (MSDN)
APIs used: QueryPerformanceCounter, QueryPerformanceFrequency,
  CreateWaitableTimerEx, SetWaitableTimer, WaitForSingleObject,
  ZwQueryTimerResolution, ZwSetTimerResolution (ntdll)
Technique: High-resolution waitable timer + busy-wait tail for
  sub-millisecond precision. This is a standard, well-known pattern.

ul_log.hpp / ul_log.cpp
------------------------
Origin: Clean-room (100% original Ultra Limiter code)
APIs used: fopen, fprintf, fflush, QueryPerformanceCounter,
  GetModuleFileNameA

ul_config.hpp / ul_config.cpp
------------------------------
Origin: Clean-room from Windows INI API documentation (MSDN)
APIs used: GetPrivateProfileStringA, GetPrivateProfileIntA,
  WritePrivateProfileStringA, GetFileAttributesA, GetModuleFileNameA
Design: Settings struct with atomic fields, preset expansion,
  INI load/save. Feature set derived from Ultra Limiter v1.0 spec.

ul_reflex.hpp / ul_reflex.cpp
------------------------------
Origin: Clean-room from NVIDIA NVAPI SDK documentation (MIT)
APIs used:
  - nvapi_QueryInterface (resolved via GetProcAddress on nvapi64.dll)
  - nvapi_interface.h (MIT) for function name -> ID mapping
  - NvAPI_Initialize, NvAPI_D3D_SetSleepMode, NvAPI_D3D_Sleep,
    NvAPI_D3D_SetLatencyMarker, NvAPI_D3D_GetLatency
  - MinHook (BSD-2) for API interception
  - DXGI vtable layout for Streamline proxy Present hook
Struct definitions: Minimal copies from nvapi.h (MIT licensed).
Hook strategy: Intercept SetSleepMode (capture + swallow), Sleep
  (suppress), SetLatencyMarker (record + forward). This is the only
  way to control Reflex pacing — any tool doing this will use the
  same three NVAPI entry points.

ul_limiter.hpp / ul_limiter.cpp
--------------------------------
Origin: Clean-room from NVAPI documentation + Windows timing APIs
Design: Frame limiter orchestrator with three pacing paths:
  1. Reflex Sleep (SetSleepMode + Sleep) — from NVAPI docs
  2. Marker-based pacing (pace on SIM_START or PRESENT markers)
  3. QPC timing fallback (hard backstop)
FG-aware rate division: target_render_fps = user_fps / fg_multiplier.
All algorithms written from the feature specification, not from any
existing implementation.

main.cpp
--------
Origin: Clean-room from ReShade addon API + ImGui API + Windows API
Components:
  - OSD: ImGui::GetForegroundDrawList + AddText/AddLine (from ImGui docs)
  - Overlay UI: ImGui widgets (from ImGui docs)
  - Monitor switching: EnumDisplayMonitors + SetWindowPos (from MSDN)
  - Fullscreen override: GetWindowLong/SetWindowLong (from MSDN)
  - Worker thread: CreateThread + WaitForMultipleObjects (from MSDN)
  - Render resolution tracking: ReShade init_resource callback
  - FG detection: GetModuleHandleW for DLL checks + FPS ratio heuristic
  - GPU latency: NvAPI_D3D_GetLatency (from NVAPI docs)
  - Crash handler: SetUnhandledExceptionFilter (from MSDN)
  - DllMain: ReShade register_addon pattern (from ReShade docs)
